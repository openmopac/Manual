(Modeling proteins)

Description and use of <file>.log

The log file summarizes the interesting features found in a protein.  Some, such as the hetero groups, are merely interesting, while others, such as charged organic sites and unusually short hydrogen bonds, often indicate problems.  The following description of the type of entries in a log file was generated from several proteins - most systems are much simpler!

Caveats, tips, and advice

There is obviously a wide range of environments that occur in proteins.  As a guide to how to handle specific types, the following points might be useful:

Important:

Check non-protein regions, in particular heme and related prosthetic groups.  In conjugated systems, the presence or absence of a hydrogen atom can alter the electronic structure completely.

Compare the sequence generated by PDBOUT to that in the PDB file.   If there are any differences, then carefully examine the offending residue. 

Advice:

When metals are present, the "TOPOGRAPHY OF SYSTEM" will likely be faulty if PDBOUT is used: metal atoms might be connected to ligand atoms, but the ligand atom might not be connected to the metal.  If this happens, and you need to use the topography, run a job that does not imply that the residue sequence is to be evaluated, i.e., avoid keywords such as RESIDUES or PDBOUT.  A suitable keyword for generating the charges is CHARGES.

The MOZYME function only allows closed-shell calculations, therefore although closed-shell atoms, e.g., Fe2+ can be run, open-shell atoms, e.g., Fe3+ cannot be run.  This issue can result in complications when changing oxidation states are involved.

When metals are present, the "TOPOGRAPHY OF SYSTEM" will likely be faulty if PDBOUT is used: metal atoms might be connected to ligand atoms, but the ligand atom might not be connected to the metal.  If this happens, and you need to use the topography, run a job that does not imply that the residue sequence is to be evaluated, i.e., avoid keywords such as RESIDUES or PDBOUT.  A suitable keyword for generating the charges is CHARGES.

When adding hydrogen atoms, do not use PDBOUT in the first calculation. If  PDBOUT is present in the first calculation, then it will create a PDB file instead of an ARC file.  Because the second calculation generates a PDB file, any PDB file created by the first calculation will be over-written.  So the only effect of having  PDBOUT in the first calculation is to suppress the creation of an ARC file.

If a problem is encountered with a PDB file, a useful approach is to prepare a "clean" PDB file, one that uses only information from the original PDB file.  To prepare a "clean" PDB file, edit the X-ray PDB file to add keywords  0SCF PDBOUT NORES and run that file. Then run a job using keywords 0SCF PDBOUT only.  Compare the PDB files that are produced by these two runs.  A useful comparison tool is WINDIFF.  Any differences need to be either explained or corrected.  Common differences are:

  • Atoms are missing from residues, resulting in their name changing.  This is usually not important. Important residues are almost always correct.

  • An "unknown" residue has a huge number of atoms.  This is most often caused by spurious close contacts giving rise to bonds.  Use CVB to break unwanted bonds.

  • Hetero groups should be numbered after the protein residue numbers.  If a hetero group has a number lower than the highest residue number, change the hetero number in the original X-ray structure to a higher value.

Less important:

Check the log file for "COVALENT BOND" If present, this normally means that a hydrogen atom is bridging between two heavy atoms.  This almost always indicates a fault in the structure. Before taking any action, however, examine the PDB file to see what is involved.  If the only fault is that the hydrogen atoms are in the wrong place, i.e., they are attached to the correct atom, but have been positioned near to another atom, causing a bridged hydrogen to exist, then ignore the fault - it will be automatically corrected when the positions of the hydrogen atoms are optimized.

Protein end groups, particularly the carboxylate end, are often faulty, e.g., -CO instead of COO.  By default, -CO is converted to CHO; to convert it to a carboxylic acid group, add an oxygen atom to the CO to form COO.
