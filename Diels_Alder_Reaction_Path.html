Diels Alder Reaction Path

This example represents the reaction path calculation of the Diels-Alder addition of ethylene, C2H4, to butadiene, CH2-CH-CH-CH2, to give cyclehexadiene. Reaction paths are mapped out starting with the transition state.  They follow the normal coordinate for reaction in one direction, and then follow the same coordinate in the reverse direction.  All the steps can be done in one calculation.  These steps are:

(A) Calculate the force constants and normal coordinates for the transition state. The normal coordinate for reaction is the first mode.  This should have an imaginary frequency, but for convenience this is normally printed as a negative frequency.  There should be exactly one of these for a transition state geometry.

 (B) Using the reaction coordinate, the transition state geometry is displaced a small amount in the direction of the positive direction of the reaction.  This moves the system in one direction, either towards the reactants or towards the products. 

(C) An IRC calculation is then started. This follows the steepest descent path in mass-weighted coordinate space, and maps out one half of the reaction.

(D) The starting geometry is displaced a small amount in the negative direction of the reaction, and a second IRC calculation run.

Because the IRC paths both start at the transition state, the first IRC path is reversed.  The second IRC path is then attached to the end of the first IRC path, to give the complete reaction path, i.e., reactants - transition state - products. 

Also, because normal coordinates are ill-defined by a phase, there is no way of knowing before the job is run whether the reaction will the the forward or the reverse reaction.  The easiest way to find out which one it is is to run the job, and if it goes reactants => products, then no further action is needed.  If it is products => reactants then replace IRC=1* with IRC=-1* and re-run.

 

Significance of keywords used in this job

HTML Write a simple HTML web-page for use by JSmol
FORCE Generate the normal modes.  Look at the output and confirm that Root 1 is negative and that all other roots are positive.
IRC=1* Generate the complete Intrinsic Reaction Coordinate for the reaction. (That's what the asterisk (*) means.) Use IRC=-1* if the reaction goes the wrong way.
CYCLES=80 For small systems, this keyword is not necessary.  For large systems, e.g., proteins, the IRC can proceed downhill for a long time, using many days of CPU time.  A useful option is to not use this keyword, and instead monitor the output.  When it looks as if the IRC is starting to wander (when the DHf stops descending), kill the job and use the results for further work.  If this strategy is used, replace IRC=1* with IRC=1, and in a separate run use IRC=-1.  Use utilities to join the two sets of results together.
