Setting up Halite to use Symmetry

Halite, NaCl, has a simple cubic structure, with the repeat unit consisting of Na4Cl4.  There is exactly one variable, this can be a Na-Cl distance, a unit cell length, or a Na-Na distance.  The large unit cell used in MOPAC calculations consists Na216Cl216 , If no symmetry were to be used, this would result in 3*2*216 - 6 = 1290 geometric variables. By using symmetry, this can be reduced to a single variable.  Several steps are required in adding symmetry to a data-set; these steps will now be described.

The starting point is the primitive Na4Cl4 cluster.  The four sodium ions form a regular tetrahedron and are positioned at the diagonal vertices of a cube, the chloride ions are at the other vertices; these also form a tetrahedron.  Each of the three equivalent translation vectors move the Na4Cl4 cluster by twice the Na-Cl distance in the direction of the Na-Cl axis. 

Symmetry can be used in either Cartesian or internal coordinates.  For most solids, including halite, internal coordinates are easier to use, particularly in those cases where there is exactly one variable.  To specify internal coordinates, MAKPOL keyword INT is used.  To automatically add symmetry, keyword SYMMETRY is used.  Because the translation vector (5.64 Å) is so small, the large unit cell to be built will use three primitive unit cells in each direction. Given this information, the data set for halite to be used by MAKPOL can be constructed, see MAKPOL data set for Halite.

When this data set is run using MAKPOL, it produces a much larger file in which most geometric quantities are symmetry related.  Several changes must still be made to the output in order to reduce the number of variables down to one.  These are:

(A) Because all angles and dihedrals are symmetry-defined, their optimization flags can all be set to zero, and all reference to them in the symmetry functions can be deleted.  These symmetry functions have the labels 2, 3, and 14 in the list of symmetry relationships at the end of the data set.

(B) All distances can be related to the first distance in the data set, here 3.988082.  Because a lot of symmetry is already present, only those distances that still have an optimization flag of "1" need to be defined.  For this, internal coordinate symmetry function 19 is used. This unique symmetry function defines a distance as a multiple of another distance.  For atom 5, a Cl, the distance 4.884383 is 1.224744871 times the first distance, so the symmetry function would be written as "2 19 1.224744871 5"  This can be read as "The distance to be used for atom 5 is 1.224744871 times the distance used for atom 2."  It might appear that 1.224744871 (nine decimals) is unduly precise given that the distances for atoms 2 and 5 are only printed to six decimals.  Precision can be increased by using a simple trick.  Look at the square of the ratio of 4.884383/3.988082 = 1.2247449.  This is 1.5000000147.  Clearly the ideal ratio is thus √(1.5).  This precise ratio can then be used in place of the approximate value obtained by comparing the distances.  Higher than normal precision is justified here because it reduces the potential for loss of symmetry due to round-off.

The optimization flag for the distance for atom 5 would then be set to zero, and the new symmetry function put at the start of the symmetry functions.  This process would be performed on all the remaining variables.  

When all the changes are made, the result would be the completely symmetry defined data set.  Note that the number of symmetry functions has decreased due to the deletion of all angular symmetry relationships.
