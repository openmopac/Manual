Preparing protein data sets

Protein specific features

Constructing data-sets for polypeptides, proteins and protein-like molecules is difficult.  Most of the sources of protein structures, such as the Protein Data Bank, do not include hydrogen atoms, or, if they are present, there are errors in where they are put.  These errors usually result in sites being ionized when they should be neutral, and vice versa.  MOPAC contains a utility for adding hydrogen atoms to protein and other organic structures.  The default is to make all organic sites neutral, so readily ionizable residues, such as Lys,  Arg, Asp and Glu are, by default, neutral.  Several utilities are provided in MOPAC that help with constructing data-sets.  Running these utilities uses very little CPU time.

Steps involved in preparing a protein data set for use in modeling protein behavior

Optimize the positions of the hydrogen atoms

Using the information in the <file>.log file, edit the <file>.arc structure to make any necessary corrections.  If no log file exists then skip this step.  Make only the absolute minimum edits necessary, i.e., limit edits to the addition or deletion of atoms, but, unless essential, don't move atoms. 

If atoms must be moved, edit the file <file_from_pdb>.pdb and save the edited file as <file>.mop, then re-run that file, in other words, start from the beginning again.  This avoids having to move hydrogen atoms, a procedure that is prone to human errors.

Save the edited file as <file_opt-H>.mop.

Next, optimize the positions of the hydrogen atoms while holding the positions of all the other atoms fixed, then check the salt bridges and other features.  If any changes are made, re-run the partial optimization to relieve stresses once more.  The resulting structure will then be a good starting point for subsequent operations.

(Optional) Resequencing the atoms to make all atoms in a residue contiguous

When hydrogen atoms are added, they are put at the end of the data-set and PDB file.  This allows a simple comparison to be made between the raw PDB file and the hydrogenated form.  For all subsequent work with proteins,  it might be easier to have all atoms in each residue together.  This can be achieved by resequencing the data set, using RESEQ.  At present, the advantages or disadvantages of doing this are not obvious.   The operation is fast and easy to carry out, so give it a try?
