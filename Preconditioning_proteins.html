Preconditioning Proteins

Introduction

The MOZYME function provides a very powerful tool for the study of proteins, in particular globular proteins.  Only systems that can be described with a closed shell wavefunction can be modeled.  In addition to proteins composed only from the standard 20 residues, more complicated proteins, such as bacteriorhodopsin, myoglobin, cytochrome-P450, and hemoglobin can also be modeled.  Organometallic enzymes, particularly those involving the heme system, are normally regarded as open shell, but their structures can apparently be represented with good accuracy using the PM6 RHF method.  No problems should occur with even quite large systems: structures of systems of almost 10,000 atoms have been optimized in a few CPU weeks.  However, problems are likely to occur in systems with significantly more than 10,000 atoms.

The most difficult operation in calculating protein structures involves preparing the data set.  None of the common sources of reference data for protein geometries are suitable for use as a data set.  This includes, in particular, the Protein Data Bank (the PDB). While all the proteins used in developing the MOZYME function were obtained from the PDB, in every case extensive work had to be done in preparing a data set that could be run using MOZYME.  Common deficiencies or features that make the raw PDB files unsuitable are:

  • For most PDB entries, hydrogen atoms are missing. 

  • There might be positional disorder (groups of atoms can occupy two possible conformations, and the PDB file records both)

  • There might be structural disorder (different residues in the same position in the sequence, e.g. ILE and LEU might both be present a fraction of the time in a given position.)

  • Residues might be missing.  (These are usually only those residues on the surface of a protein that are not involved in activity, and are not tightly bound.)

  • Exotic atoms might be present, e.g. Hg, Sm, or La.

All these features and faults must be addressed before a MOZYME run can be started.

Recommended Sequence for Preconditioning a Protein Data Set

1. Before starting, get a copy of the data set representing the protein of interest; the PDB is an excellent source for such data files.

2. The first operation is to resolve positional and structural disorder.  Most GUIs that can handle PDB files will be able to do this when the data set is read in.  Look at the data set.  If there are gaps in the residue sequence, find out whether these represent genuine gaps or represent missing data.  Normally, gaps are not very important, and can safely be ignored. The rare case is when there are one or more residues missing inside the protein - this normally would require the missing residues to be replaced.

3. If necessary, add hydrogen atoms.  Even if the protein already has hydrogen atoms, quite often several of them are missing.  During this operation, several problems can occur. The commonest complications are:

  • Too many or too few hydrogen atoms are added to a heavy atom resulting in the wrong saturation.  Thus if a phenyl group has excess hydrogen atoms, it might be converted to a cyclohexane.

  • Check hetero groups carefully.  Thus heme should have the correct porphyrin ring structure, and small hetero groups, such as carbon monoxide and molecular oxygen, should not have "extra" hydrogen atoms.

4. Prepare a MOPAC data set; use Cartesian coordinates.  In the first run, use keywords 0SCF and PDBOUT or HTML.  Inspect the resulting structure using JSmol or other GUI.  There should be no unknown residues.  If there are, and you're sure that they are correct, use keyword XENO to re-define the unknown residue or heterogroup. This step is not essential, but if it's done then potential problems associated with the unusual residue can be more rapidly located, if it's not done, the unknown residue or heterogroup will be indicated as RES or UNK

5. Prepare a new data set.   Run this set using CUTOFF=6, GNORM=30, NOOPT, and OPT-H.  That will optimize the positions of all the hydrogen atoms.

6. After the job has run, use the .arc file to make a new data set.  Run this new set with keywords 0SCF and PDBOUT.  Examine the resulting file to see if any hydrogen atoms have migrated to form salt bridges.  If they have, examine the bridge to see if it is chemically sensible.

7. If everything is still okay, edit the data set to change all optimization flags from 0 to 1, and re-run with keywords CUTOFF=6 GNORM=30 T=1w plus a CVB or SETPI keyword, if necessary.

8. If desired, add all the PDB preamble (everything in the PDB file from "HEADER" to the line before the first atom) to the data set as comments.  Put an asterisk at the start of each comment to indicate that it is a comment.  If desired, add the keyword  START_RES(text).  If the preamble, and START_RES(text) if necessary, is added, then when PDBOUT is used, the resulting file will be almost indistinguishable from a regular PDB file.

9. Once the job finishes, edit the ARC file to make a new data set with keywords  GNORM=10 T=2w plus a CVB or SETPI as necessary, and re-run.

10. Edit the ARC file to make a preconditioned starting MOPAC data set.  This can be used for following a reaction path, finding a transition state, etc.
